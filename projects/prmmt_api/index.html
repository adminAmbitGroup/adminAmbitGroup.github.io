Overview

This documentation specifies the organizational structure, naming conventions and API for best coding practices within the Prmmt department when working in ServiceNow. This ensures consistent and accurate references to any apps that are developed. As a general rule please try to follow the conventions below.

Coding practices

Servicenow uses both global as well as local application scopes to share and separate concerns from one app to the other. Because of this, you should use the naming conventions described below.

File Naming

When adding files(including content blocks) to apps, you are expected to follow these guidelines

  • Global
    • Keyword Prefix: Prmmt
      • ***Global Script UI do not need Prefix***
    • PascalCase – Start with uppercase
    • Use underscores and no spaces in between words
    • Used for all global blocks, script names, macro,etc.
    • Examples:Prmmt_Tbl_Script_Inc, Prmmt_Static_Block, Prmmt_Dyn_Block
  • Local
    • Keyword Prefix:
      • Grants Bi: Gbi
      • QASP: Qasp
      • CCMS: Ccms
      • Tagr: Tagr
      • ***Local Script UI will add prefix app scope name before file.***
    • PascalCase – Start with uppercase
    • Use underscores and no spaces in between words
    • Used for all local blocks, script names, macro,etc.
    • Examples:Gbi_Tbl_Script_Inc, Qasp_Static_Block, Ccms_Dyn_Block
