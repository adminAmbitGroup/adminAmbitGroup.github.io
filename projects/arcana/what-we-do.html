Business Understanding

Business Understanding phase for data analytics projects involve determining and understanding the business objectives or opportunity and planning activities to set the foundation for initiating a project.

1. Understand business objectives

  • Define the opportunity or objectives that describe the projectâ€™s purpose
  • Identify expected results
  • Assign team

Key Questions

  • What is the business objective?
  • What are the pain points, if any?
  • What needs will this service address?
  • Who will be the primary users of this service?
  • Is this service replacing an existing tool, report(s) and/or process?

2. Understand scope of service

  • Understand scope and resource needs
  • Draft project plan

Key Questions

  • Understand scope and resource needs
  • Are there reference documents?
  • Are the sources of data known?
  • Who are the points-of-contact for additional information?
  • What are the risks and assumptions with the project?
  • What are the milestones for the project?

3. Assign team

  • Define roles and responsibilities. Assign one leader to be held accountable for the success or failure of the overall service.
  • Set up rules of engagement, such as internal and external communication
  • One of the roles should include a product owner who is ultimately responsible for how well the service meets the needs of its users.

Key Questions

  • What are the rules of engagement between members of the team, such as communications, versioning, etc...?

Data Understanding

Data Understanding phase for data analytics projects involve initial data collection and activities that enable the project team to become familiar with the data, identify data quality problems, and discover first insights into the data. Additionally, this stage also involves defining requirements based on the scope identified during the Business Understanding phase into detailed, testable requirements and converting them into design specifications that developers use to implement (e.g. code) the solution.

4. Capture requirements iteratively

  • Define requirements and ensure that they are well defined and testable

Key Questions

  • What are available materials relevant to project?
  • What primary tasks are the users trying to accomplish?
  • Where does this specific project fit into the larger way people currently obtain the service being offered?
  • Are there alternative solutions to this service? What are the pros and cons with each alternative solution?
  • What metrics will best indicate how well the service is working for its users?
  • How does the service's design visually relate to other government services?

5. Understand data

  • Familiarize with data
  • Collect initial set of data
  • Understand data quality problems

Key Questions

  • Is the data needed for the service available and accessible?
  • Is the available data accurate, complete, and current?

Data Preparation and Modeling

Data Preparation and Modeling phase for data analytics projects involve activities for (and in support of) converting design specifications into executable programs or solutions using approved tools and products. Therefore, this stage also involves preparing the dataset(s) used for modeling or for analysis work, as well as any set up needed for enabling code development (or construction) in an optimal manner, such as the setup of developer workspaces.

6. Prepare data

  • Set up developer workspace

Key Questions

  • What is the source code version control system to use?
  • Does the entire project team have access to issue tracking and version control systems?
  • What are the environments?
  • What are the sources of data?
  • Where is the data hosted?
  • Where is the data hosted?
  • How are developer tasks assigned and managed?
  • How does the team communicate?

7. Build the service

  • Use code reviews to ensure quality
  • Ensure that the individuals building the service communicate closely
  • Conduct integration testing, if appropriate

Key Questions

  • How are defects tracked?
  • How do you collect user feedback during development?
  • How is that feedback used to improve the service?
  • How is the feature (requirements) backlog managed? What tool is used?
  • Where are test results stored?

Evaluation

Evaluation phase for data analytics projects involves conducting both targeted testing (i.e. testing of changes within the scope of requirements) and regression testing (i.e. testing of functionality expected to be unaffected by the change scope). Testing may be executed by the technology team, business or both teams. For testing conducted by the technology team, the Project Manager must ensure proper segregation of duties standards are adhered to (i.e. developers should not conduct QA testing and User Acceptance Testing).

8. Conduct QA testing

  • Set up/prepare test environment
  • Migrate code to test environment
  • Seed test environment with test data
  • Develop a plan for conducting QA testing, to include test cases with expected results whenever possible
  • Establish Requirements Traceability Matrix linking test cases to requirements
  • Conduct QA testing

Key Questions

  • How are findings reported?
  • How can the team find out the status?
  • When should regression testing be conducted instead of just targeted testing?

9. Conduct User Acceptance Testing

  • Develop a plan for conducting User Acceptance testing, to include test cases with expected results whenever possible
  • Conduct UAT
  • Check usability of service for simplicity and intuitiveness

Key Questions

  • Does service meet business objectives?

Deployment

The Deployment phase accomplishes the implementation planning and deployment or installation of the certified code baseline into the production environment. Of note, implementation planning may begin far in advance of the actual production deployment activities. This process also includes post-implementation review that provides insight into capability deficiencies, process inefficiencies, and process deterioration.

10. Prepare for production deployment

  • Set up/prepare test environment
  • Migrate code to test environment
  • Seed test environment with test data
  • Develop a plan for conducting QA testing, to include test cases with expected results whenever possible
  • Establish Requirements Traceability Matrix linking test cases to requirements
  • Conduct QA testing

Key Questions

  • Are release notes prepared? Release notes document describes the scope/functionality of the release to be deployed into production.
  • Is the status of all requirements up to date?
  • All all design and test artifacts up to date?
  • Did the product owner certify the release for production deployment?
  • Are job aids needed?

11. Conduct deployment

  • Conduct quick "shake out", which involves end-to-end testing to ensure that the environment does not crash.
  • Conduct training, if needed.

Key Questions

  • What is the mechanism to receive user feedback regarding new release?
  • What is change management process to prioritize change requests?
  • What is process to measure performance of service against stated objectives?
